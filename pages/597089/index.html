# @font face

@font-face {
  font-family: 'fontName';
  src: url('font.eot');
  src: url('font.eot?#iefix') format('embedded-opentype'),
         url('font.woff2') format('woff2'),
         url('font.woff') format('woff'),
         url('font.ttf') format('truetype'),
         url('font.svg#ciclefina') format('svg');
  font-weight: normal;
  font-style: normal;
}

// 简化写法，ie9可以使用woff字体
@font-face {
    font-family: 'fontName';
    src: url(font.eot);
    src: local('☺'), url(font.woff2) format('woff2'),url(font.woff) format('woff'),url(font.ttf);
}
  • font-family：指定字体的名称。
  • src：
    • svg格式是为了兼容ios4.1及之前版本，可舍弃。
    • eot: ie6-ie8仅支持这种格式。
    • woff是专门为web开发设计的，Android4.4全面支持。
    • woff2: web开发第一首选字体。
    • ttf: 老版Android支持这种格式，可舍弃。
    • ?#iefix: ie9之前当src包含多个url时，会把整个当做一个地址解析，加上?后，?后面的会当做url参数。
    • 两个src: 兼容模式的ie7和ie8不认识?解决方案。
    • format: 提前让浏览器知道字体的格式。
  • font-weight/font-style: 用来设置对应字体样式或字重下该使用什么字体。参见@font-face tip: define font-weight and font-style to keep your CSS simple | 456 Berea Street (opens new window)
@font-face {
    font-family: quote;
    src: local("SimSun");
    unicode-range: U+201c,U+201d;
}
.font {
    font-family: quote, "Miscrosoft Yahei";
}

  • unicode-range: 让特定字符或特定范围的字符使用指定字体。统一码_百度百科 (baidu.com) (opens new window)

    html: &#x加上Unicode编码

    js: \u加上Unicode编码

    css: \加上Unicode编码

    unicode-range: U+加上Unicode编码

@font-face {

    src: url(iconfont.eot?#iefix) format("embedded-opentype"),url(iconfont.woff) format("woff"),url(iconfont.ttf) format("truetype"),url(iconfont.svg#iconfont) format("svg")
}

# font-family

网站字体：

  • 网页安全字体
  • 跟随系统
body {
    font-family: -apple-system,BlinkMacSystemFont,"Segoe UI","Noto Sans",Helvetica,Arial,sans-serif,"Apple Color Emoji","Segoe UI Emoji";
}
  • -apple-system只在macOS中有效，是system-ui字体族还没出现之前的一种私有语法，可以让Firefox浏览器和Safari 9.1～Safari 10.1浏览器使用系统字体。
  • BlinkMacSystemFont也只在macOS中有效，是Chrome 53～Chrome 55浏览器使用系统字体的一种非标准语法。考虑到目前system-ui字体族的兼容性，并从适应未来的角度来看，BlinkMacSystemFont可以删掉了。
  • Segoe UI、Helvetica、Arial是给不支持系统字体的浏览器兜底用的，如IE浏览器、Edge浏览器等。其中需要注意以下3点。
  • Segoe UI是Windows操作系统从Vista版本开始默认的西文字体族，可以在Windows操作系统上以最佳的西文字体显示。
  • Helvetica是macOS和iOS中很常用的一款无衬线字体。
  • Arial是全平台都支持的一款无衬线字体，可以作为最后的兜底，例如较老版本的Windows操作系统。
  • Apple Color Emoji和Segoe UI Emoji是emoji字体
@font-face {
    font-family: Emoji;
    src: local('Apple Color Emoji'), local('Segoe UI Emoji'), local('Segoe UI Symbol'), local('Noto Color Emoji');
    unicode-range: U+1F000-1F644, U+203C-3299;
}
body {
    font-family: system-ui, -apple-system, Segoe UI, Roboto, Emoji, Helvetica, Arial, sans-serif;
    -webkit-font-smoothing: antialiased;
    -moz-osx-font-smoothing: grayscale;
    -webkit-tap-highlight-color: transparent;
}
  • Apple Color Emoji用在Apple的产品中的，如iPhone（iOS）或者Mac Pro（macOS）等
  • Segoe UI Emoji是用在Windows操作系统中的emoji字体。
  • Segoe UI Symbol是在Windows 7操作系统中添加的一种新字体，是一种Unicode编码字体，显示的是单色图案，非彩色图形。
  • Noto Color Emoji是谷歌的emoji字体，用在Android和Linux操作系统中。
  • -webkit-font-smoothing: antialiased; -moz-osx-font-smoothing: grayscale; -- 抗锯齿，参见字体平滑 - CSS（层叠样式表） | MDN (mozilla.org) (opens new window)
  • -webkit-tap-highlight-color: transparent; 去掉点击链接时高亮。-webkit-tap-highlight-color - CSS（层叠样式表） | MDN (mozilla.org) (opens new window)

# 字体其他属性

  • font-variant (opens new window): 在小型大写字母和普通文本选项之间切换。
  • font-kerning (opens new window): 开启或关闭字体间距选项。
  • font-feature-settings (opens new window): 开启或关闭不同的 OpenType (opens new window) 字体特性。
  • font-variant-alternates (opens new window): 控制给定的自定义字体的替代字形的使用。
  • font-variant-caps (opens new window): 控制大写字母替代字形的使用。
  • font-variant-east-asian (en-US) (opens new window): 控制东亚文字替代字形的使用，像日语和汉语。
  • font-variant-ligatures (opens new window): 控制文本中使用的连写和上下文形式。
  • font-variant-numeric (opens new window): 控制数字，分式和序标的替代字形的使用。
  • font-variant-position (opens new window): 控制位于上标或下标处，字号更小的替代字形的使用。
  • font-size-adjust (opens new window): 独立于字体的实际大小尺寸，调整其可视大小尺寸。
  • font-stretch (opens new window): 在给定字体的可选拉伸版本中切换。
  • text-underline-position (opens new window): 指定下划线的排版位置，通过使用 text-decoration-line 属性的underline 值。
  • text-rendering (opens new window): 尝试执行一些文本渲染优化。
