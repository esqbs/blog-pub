浏览器工作原理

参考渲染页面：浏览器的工作原理 - Web 性能 | MDN (mozilla.org) (opens new window)

等待资源加载时间和大部分情况下的浏览器单线程执行是影响 Web 性能的两大主要原因

# 请求

# DNS查询

浏览器向名称服务器发起 DNS 查询请求，最终得到一个 IP 地址。

第一次请求之后，这个 IP 地址可能会被缓存一段时间，加速后续的请求。

优化

  1. 对跨源资源使用dns-prefetch，参考使用 dns-prefetch - Web 性能 | MDN (mozilla.org) (opens new window)

    <link rel="dns-prefetch" href="https://fonts.googleapis.com/" />

# HTTP请求

优化

  1. 减少 HTTP 请求的数量，合并请求
  2. 使用 HTTP/2 协议提高传输的效率

# TCP握手

tcp三次握手

# TLS协商

深入浅出 HTTPS (详解版) - huansky - 博客园 (cnblogs.com) (opens new window)

图解 ECDHE 密钥交换算法 - 小林coding - 博客园 (cnblogs.com) (opens new window)

# HTTP响应

优化

  1. gzip资源压缩，包括js、css、图片等，CompressionWebpackPlugin (opens new window)
  2. 缓存，包括http缓存、server worker、CDN
  3. 懒加载 (opens new window)

# 解析

# 构建DOM树

请求页面的 HTML的前 14KB 中包含浏览器开始渲染页面所需的所有内容，或者至少包含页面模板（第一次渲染所需的 CSS 和 HTML）对于 web 性能优化来说是重要的

优化

  1. 尽量减少页面的DOM数量
  2. 提供不同分辨率的图片资源，图片懒加载
  3. 除非非常必要，否则不要使用嵌入式 <iframe>
  4. 使用 rel="preload" 预加载内容

# 构建CSSOM树

CSS 被视为渲染阻塞 (opens new window)资源，在 CSSOM (opens new window) 构造完成之前，浏览器不会渲染任何已处理的内容

优化

  1. 删除不必要的样式，PurgeCSS (opens new window)
  2. 压缩CSS，MiniCssExtractPlugin (opens new window)
  3. 简化选择器
  4. 使用 CSS 精灵图减少图像相关的 HTTP 请求
  5. 将 CSS 拆分为独立模块，使用媒体查询加载
  6. 预加载重要资源
  7. 字体优化
  8. 图片优化 (opens new window)
  9. 视频优化 (opens new window)

# 渲染

# 样式

将 DOM 和 CSSOM 组合成渲染树

# 布局

第一次确定节点的大小和位置称为布局。随后对节点大小和位置的重新计算称为回流

# 绘制

为了确保平滑滚动和动画，占据主线程的所有内容，包括计算样式，以及回流和绘制，必须让浏览器在 16.67 毫秒内完成

绘制可以将布局树中的元素分解为多个层。将内容提升到 GPU 上的层（而不是 CPU 上的主线程）可以提高绘制和重新绘制性能，包括 video (opens new window) 和 canvas (opens new window)，任何 CSS 属性为 opacity (opens new window) 、3D transform (opens new window)、filter (opens new window)、will-change (opens new window) 的元素，还有一些其他元素

优化

  1. 使用transform、opacity、filter实现动画
  2. 合理使用will-change或tanslateZ提升元素到合成层

# 合成

分层确实可以提高性能，但是它以内存管理为代价，因此不应作为 web 性能优化策略的一部分过度使用

# 交互

# javascript

JavaScript 会被解析、编译和解释，脚本被解析为抽象语法树

有些浏览器引擎会将抽象语法树 (opens new window)输入编译器，输出字节码

优化

  1. 脚本标签上使用 defer (opens new window) 或 async (opens new window) 属性，脚本尽量放在文档底部
  2. 对大的js文件进行分包处理，分包优化splitchunksplugin
  3. 对暂时不需要的js异步加载
  4. 体积巨大，持续时间长的js代码块使用 Web workers (opens new window)
