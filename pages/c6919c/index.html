Maven

# 一、Maven简介

Maven是一个基于项目对象模型（POM）用于进行项目的依赖管理、生命周期管理的工具软件

  • 项目的依赖（jar）管理
  • 项目生命周期管理
  • 聚合功能

# 二、Maven安装

  1. Maven下载 [下载地址](Maven – Download Apache Maven (opens new window))，Maven 是一个基于 Java 的工具，所以要做的第一件事情就是安装 JDK

  2. Maven安装

    • 解压
    • 目录结构
      • bin 存放指令文件夹（Maven提供了一个mvn指令）
      • boot 包含一个类加载框架的jar文件
      • conf 包含Maven的核心配置文件setting.xml
      • lib 存放Maven运行所需的jar文件

  3. 配置环境变量

    • MAVEN_HOME C:\java\apache-maven-3.8.5 (maven文件地址)
    • Path %MAVEN_HOME%\bin

# 三、Maven项目结构

  1. 项目结构
project(项目名称)
  --src
    --main(存放项目源代码)
 	  --java(存放java代码，相当于传统项目中的src目录)
      --resources(存放配置文件和静态资源的目录，相当于传统项目中的web目录)
    --test(存放项目的单元测试代码)
  --pom.xml

  1. pom.xml

    Maven可以根据pom文件的配置对此项目进行依赖管理

<?xml varsion="1.0" encoding="utf-8"?>
<!--.xsd 语法标准-->
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <!--指定项目模型版本-->
    <modelVersion>4.0</modelVersion>
    <!--指定项目标识：G 企业标识 A 项目标识 V 版本-->
    <!--公司或者组织的唯一标志，并且配置时生成的路径也是由此生成， 如com.companyname.project-group，maven会将该项目打成的jar包放本地路径：/com/companyname/project-group-->
    <groupId>com.companyname.project-group</groupId>
    <artifactId>project</artifactId>
    <version>1.0.0</version>
    
    <!--打包成war包-->
    <packaging>war</packaging>
    <!--配置当前项目的依赖, mvn test 可下载依赖-->
    <dependencies>
        <dependency>
            <groupId>mysql</groupId>
            <artifactId>mysql-connector-java</artifactId>
            <version>8.0.27</version>
        </dependency>
    </dependencies>
</project>

# 四、配置仓库

  1. 配置本地仓库
<localRepository>/path/to/local/repo</localRepository>
  1. 配置公共仓库
<mirror>
    <id>aliyunmaven</id>
    <mirrorOf>*</mirrorOf>
    <name>aliyun</name>
    <url>https://maven.aliyun.com/repository/public</url>
</mirror>

# 五、项目生命周期管理

项目构建的生命周期：项目开发完成后部署到运行环境的过程

  • mvn clean 清除缓存
  • mvn validate 检查
  • mvn compile 编译
  • mvn test 测试（执行maven项目中test目录下的单元测试）
  • mvn package 打包（war 、jar）
  • mvn install 安装（jar被安装到本地仓库）
  • mvn deploye 部署（将项目生成包放到服务器）
